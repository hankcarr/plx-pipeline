# PCB Footprint Standardization Pipeline

## Overview

A tool to download PCB component footprints from various sources, synthesize a
standardized footprint using Brant and Graham's method, and import the result
into Pulsonix for use in PCB design.

## Team

- **Hank** — project lead, Pulsonix user
- **Brant (Frack)** — programming/code review; currently performs synthesis manually
- **Graham (Frick)** — hardware/schematics
- **Target fab** — PCBway

---

## Synthesis Method

### Brant's Rule
For each pad, take the **maximum extent** across all source footprints. The
result is a footprint that every source variant fits within.

### Constraints
Pad expansion is bounded by PCBway's manufacturing design rules. If expanding
a pad would violate minimum spacing, the conflict is flagged for manual review
rather than silently producing an unmanufacturable footprint.

---

## PCBway Design Rule Limits

| Parameter                  | Absolute Minimum  | Recommended       |
|----------------------------|-------------------|-------------------|
| Min trace width            | 0.1mm / 4mil      | 0.15mm / 6mil     |
| Min spacing                | 0.1mm / 4mil      | 0.15mm / 6mil     |
| Min drill size (CNC)       | 0.2mm             | ≥0.3mm            |
| Finished hole diameter     | 0.2mm – 6.2mm     | —                 |
| Drill size tolerance       | ±0.08mm           | —                 |
| Min annular ring width     | 0.15mm / 6mil     | —                 |
| Min silkscreen char height | 0.8mm             | —                 |
| Min silkscreen char width  | 0.15mm            | —                 |

> Note: Holes smaller than 0.3mm or larger than 6.3mm incur extra charges.
> Strongly recommended to design traces and spacing at 0.15mm / 6mil or above
> to avoid additional cost.

## Pipeline Steps

1. **Input** — multiple footprint files for the same component from various
   sources (SnapEDA, Ultra Librarian, Samacsys, manufacturer direct, etc.)
2. **Parse** — extract pad geometry from each source footprint
3. **Synthesize** — union/max extents per pad; flag any spacing violations
   against PCBway DRC limits for manual review
4. **Standardize** — apply Brant and Graham's standard layer set
5. **Output** — in a format Pulsonix can import

## Format Considerations

### Best candidate for intermediate format
**KiCad `.kicad_mod`** — plain text S-expression format, well documented,
readable and writable without tooling.

### Pulsonix import — confirmed formats
Pulsonix accepts the following relevant formats (ASCII required):

| Format | Notes |
|---|---|
| Eagle `.lbr` | Versions 4.09 to current; XML/ASCII; **recommended target** |
| PLX | Pulsonix native footprint format; ideal if spec is available |
| DesignSpark | Already in Hank's toolchain |
| Gerber 274D/274X | Lossy; intelligent rebuild only; not suitable for footprints |
| CSV | Component placement / netlist / tech rules |
| STEP `.stp` | 3D board profiles only |

### Recommended output format
**Pulsonix Library Toolkit format** — Pulsonix's own native integration
format. Keyword-based, comma-delimited, one command per line. Imported
directly via Library Manager. Fully documented in the Pulsonix Library
Toolkit manual. No third-party tool required.

This is preferred over PLX, which is generated by PCB Libraries Footprint
Expert (third-party) and requires a `.ptf` technology file and layer mapping
file on import. The Library Toolkit format is self-contained.

Eagle `.lbr` remains a fallback if the Library Toolkit format proves
problematic in practice.

### Key Library Toolkit keywords (confirmed)
| Keyword | Purpose |
|---|---|
| `FORMAT` | First line of file; declares type (footprint) and version |
| `FOOTPRINT` | Declares footprint name; triggers generation |
| `ORIGIN` | Sets footprint origin position |
| `ADDPAD` | Adds a pad relative to previous pad |
| `PADWIDTH` | Pad width |
| `PADSHAPE` | Pad shape (RECT, ROUND, OVAL, etc.) |
| `PADSTYLE` | Named pad style |
| `PADLAYER` | Layer name for pad |
| `DRILLDIA` | Drill diameter for through-hole pads |
| `UNITS` | Coordinate units (mm or thou) |
| `TECHNOLOGYFILE` | Technology file to apply on import |

Full keyword reference: Pulsonix Library Toolkit manual
(pulsonix.com/downloads/manuals/12/Pulsonix%20Library%20Toolkit.pdf)

## PLX Format Specification

### Overview
PLX is a **nested S-expression format** (parenthesis-based, similar to KiCad).
Text-based, human-readable, import-only in Pulsonix (no export). Generated by
SnapEDA, PCB Libraries Footprint Expert, and other partners.

Header line: `"PULSONIX_LIBRARY_ASCII" "licence string"`

### Top-level structure
```
(library Library
  (padStyleDef ...)     ; named pad styles — defined once, referenced by name
  (textStyleDef ...)    ; named text styles
  (patternDef ...)      ; PCB footprint: pads + layer graphics
  (symbolDef ...)       ; schematic symbol with pins
  (compDef ...)         ; component: links symbol to pattern, pin mapping
)
```

### Units
Declared in `asciiHeader`: `(fileUnits MIL)` or `(fileUnits MM)`.
All coordinates and dimensions use the declared units throughout.

### padStyleDef
Defines a named pad style. Referenced by pads in `patternDef`.
```
(padStyleDef "STYLENAME"
  (holeDiam 39.3701)          ; drill diameter (0 for SMD)
  (padShape
    (padShapeType Rect)       ; Rect | Oval | Round | etc.
    (layerNumRef 1)           ; layer number (see layer map below)
    (shapeWidth 59.3701)
    (shapeHeight 59.3701)
  )
  ; repeat padShape for each layer this pad appears on
)
```
Convention: pin 1 uses `Rect` shape, all others use `Oval`/`Round`.

### patternDef — pads
```
(patternDef "FOOTPRINT_NAME"
  (originalName "FOOTPRINT_NAME")
  (multiLayer
    (pad
      (padNum 1)
      (padStyleRef STYLENAME)
      (pt x, y)
      (rotation 0)
    )
    ; ... one pad entry per pin
  )
```

### patternDef — layer graphics
```
  (layerContents
    (layerNumRef 30)          ; silkscreen, assembly, courtyard etc.
    (line (pt x1 y1) (pt x2 y2) (width w))
  )
  (layerContents
    (layerNumRef 30)
    (arc (pt cx cy) (radius r) (startAngle 0) (sweepAngle 360) (width w))
  )
  (layerContents
    (layerNumRef 18)
    (attr "RefDes" "RefDes"
      (pt x y) (rotation 0) (textStyleRef "STYLENAME") (isVisible True)
    )
  )
```

### symbolDef
Schematic symbol with pin definitions. Each pin has number, position,
rotation, length, and name.

### compDef
Links symbol to pattern. Contains `padPinMap` mapping pad numbers to
component pin references. Also carries attributes: MANUFACTURER,
PARTREV, STANDARD, etc.

### Additional format notes (from MEM2075 and Pulsonix.PLX samples)

**SMD pads** — `holeDiam 0` indicates SMD. Bottom copper layer (16) gets
`shapeWidth 0` / `shapeHeight 0` (zero size = not present on that layer).
SMD pad styles also include `startRange` / `endRange` fields (value 1/2 —
likely indicating the layer span).

**NPTH mounting holes** — `isHolePlated False` flag on the padStyleDef.

**Polygon fills** — `(poly (pt x y) ...)` in layerContents for filled areas
(e.g. component body outlines on assembly layer).

**Inline vs. expanded syntax** — both are valid. SnapEDA uses expanded
multi-line style; Ultra Librarian uses compact single-line style. Parser
must handle both.

**Multiple patternDefs per file** — Pulsonix.PLX contains three variants of
the same footprint (nominal, maximum, least) in one file under one library.
This is the IPC-7351 land pattern density approach (B = nominal, M = max,
L = least). Our pipeline should handle multi-pattern files.

**Multiple compDefs** — one compDef can reference multiple patternDefs via
multiple `attachedPattern` blocks.

**`+vbcrlf` token** — seen in Ultra Librarian output as a line continuation
artifact. Parser must tolerate and ignore it.

**Header line variants:**
- SnapEDA: `"PULSONIX_LIBRARY_ASCII" "SnapEDA ...licence..."`
- Ultra Librarian: `PULSONIX_LIBRARY_ASCII "path/to/file.PLX"` (no outer quotes)
Both are valid; parser must handle both.

**text element** — `(text (pt x y) "string" (textStyleRef "...") (isVisible bool))`
Used for pin-1 markers and other annotations directly in layerContents.

### Layer number mapping (consolidated from all samples)

Layer numbers appear to be **fixed Pulsonix standard numbers**, consistent
across SnapEDA, Ultra Librarian, and other sources. Confirmed mapping:

| layerNumRef | Confirmed use | Brant layer | Source |
|---|---|---|---|
| 1 | Copper top | TOP | All samples |
| 16 | Copper bottom (zero size on SMD) | BOTTOM | All samples |
| 18 | Assembly drawing top / RefDes / Type attrs | Assembly Drawing Top | All samples |
| 19 | Pin-1 indicator circle (TH connector) | Silkscreen Top (tentative) | RY611024 |
| 20 | Solder mask top | Solder Mask Top | All SMD samples |
| 21 | Solder mask bottom | Solder Mask Bottom | LCD2X16 |
| 22 | Paste mask top | Paste Mask Top | All SMD samples |
| 30 | Silkscreen top outlines / pin-1 dot | Silkscreen Top | LCD2X16, MEM2075 |
| 39 | Documentation / courtyard / hidden attrs | Documentation | All samples |

> **Note:** Layer numbers are fixed across all observed sources (SnapEDA,
> Ultra Librarian, Accelerated Designs). We can hardcode the layer map.
>
> **Still unobserved:** Bottom silkscreen, bottom paste mask, bottom assembly
> drawing, placement area top/bottom. All samples so far are top-side SMD or
> through-hole. A bottom-side SMD component would likely reveal these.
>
> **Layer 19 tentative** — only one occurrence, a circle arc on a TH connector.
> Could be silkscreen top (like 30) or a separate pin indicator layer. Confirm
> with Brant.

### Layer mapping notes
When importing source footprints (KiCad, Eagle, etc.), layers must be mapped
to this set. Common mappings:

| Source layer | → Brant layer |
|---|---|
| F.Cu / Top copper | → TOP |
| B.Cu / Bottom copper | → BOTTOM |
| F.SilkS / tPlace | → Silkscreen Top |
| B.SilkS / bPlace | → Silkscreen Bottom |
| F.Paste / tCream | → Paste Mask Top |
| B.Paste / bCream | → Paste Mask Bottom |
| F.Mask / tStop | → Solder Mask Top |
| B.Mask / bStop | → Solder Mask Bottom |
| F.Fab / tDocu | → Assembly Drawing Top |
| B.Fab / bDocu | → Assembly Drawing Bottom |
| F.Courtyard / tRestrict | → Placement Area Top |
| B.Courtyard / bRestrict | → Placement Area Bottom |
| Edge.Cuts / Dimension | → Outline |
| Holes / Drills | → Holes Top / Holes Bottom |
